## Challenges & patterns (no blame)

1. **Cross-cutting correctness** — Fee percentages (2.5) and OG URL correctness (2.6) both needed explicit handling of edge cases and deployment assumptions (`PUBLIC_APP_ORIGIN`, reverse proxy for `/league/*`).
2. **Scale deferred by design** — Browse uses `findMany` without pagination (2.4 retro notes); acceptable for now, risk when the catalog grows.
3. **Data model clarity** — Duplicate `contract_address` / ambiguous `findFirst` called out for leagues (2.5 follow-ups); same class of issue affects OG and JSON detail.
4. **Polish debt** — Detail page `window.alert` for connected “Enter” placeholder; ENS-only creator naming on some chains — documented for later.
5. **OG stack split** — Vite middleware + Fastify HTML path must stay aligned in dev; production routing is an ops checklist, not only code.
