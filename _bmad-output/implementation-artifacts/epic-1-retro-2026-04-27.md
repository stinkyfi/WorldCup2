- Sonic explorer verify wiring, Slither vs crytic-compile on HH3, and NodeNext `.js` import rules showed up as **process friction**, not team error. CI encodes the Slither escape hatch with a **notice** so `main` stays mergeable while upstream catches up — a documented system choice (Story 1.10 / 1.6 notes).

**Testing craft**

- **Dana (QA Engineer):** “Story 1.3’s dev notes call out **block-scoped event assertions** as a lesson from 1.2. That’s the kind of pattern reuse we want on long contract epics.”

- League: unused immutables / `_entrantsList` write path deferred for later stories.
- Contracts `package.json`: semver `^` ranges flagged as supply-chain policy decision for a future **pinning / audit** gate.

## Challenges & growth areas

1. **External toolchain drift** — Hardhat 3 and Slither integration required explicit docs and CI branching behaviour rather than a single green path.
2. **Conscious deferrals** — Several security-adjacent items are **owner-trust** or **staging-only** by design until hardening epics; the risk is forgetting to re-triage them before production scale.
3. **Multi-repo coordination** — Changes that touch `contracts/` + workflows + `deployments/` need a single owner per release train to avoid desync.
